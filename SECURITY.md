# Security Policy

## Reporting a Vulnerability

Please do not open public issues for suspected credential leaks, exposed customer data, authentication bypasses, or other sensitive security reports.

Report vulnerabilities privately through GitHub's private vulnerability reporting feature if it is enabled for this repository. If private reporting is unavailable, contact the repository owner directly and include:

- a short description of the issue,
- affected files, routes, or configuration,
- steps to reproduce when safe to share,
- whether any real service credentials, Search Console data, PostHog data, or Linear data may be exposed.

Do not include live API keys, service account private keys, session tokens, raw user identifiers, or customer analytics exports in a public report.

## Supported Versions

This project is currently maintained from the `main` branch. Security fixes are handled there first.
